# Sport Challenge Bot v11.4

Telegram-бот для нескольких групп с универсальными упражнениями и отдельными правилами каждого челленджа.

## Что нового в v11.4

- Исправлено сохранение результатов: кнопка больше не молчит при потере FSM-состояния.
- Добавлена явная транзакция SQLite, WAL и ожидание блокировки базы для Railway.
- При повторном редактировании сохраняется полный набор упражнений, включая нули.
- Ошибки сохранения показываются пользователю и записываются в Railway Logs.
- После успешного сохранения кнопки подтверждения удаляются, чтобы исключить повторные нажатия.

## Возможности v11.1

- Кнопка `📋 Правила` в личном и групповом интерфейсе.
- Шаблоны правил:
  - классический — баллы пропорционально выполнению нормы;
  - выполнил / не выполнил — баллы только за полную норму;
  - ступенчатый — 50% нормы дают половину баллов, 100% дают все баллы;
  - собственные настройки.
- В собственных настройках выбираются:
  - способ начисления баллов;
  - учёт перевыполнения;
  - критерий успешного дня;
  - количество прошлых дней, доступных для исправления;
  - порядок вступления участников.
- Правила копируются вместе с челленджем.
- После создания условия фиксируются и не меняют старые результаты.
- Старая база v10/v11 мигрирует автоматически.

## Railway

Переменные:

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_PATH` — путь к SQLite, по умолчанию `sport_challenge.db`.
- `TIMEZONE` — по умолчанию `Europe/Moscow`.
- `ADMIN_IDS` — Telegram ID администраторов через запятую, необязательно.

Команда запуска уже указана в `Procfile`.

## Обновление

1. Сделайте резервную копию SQLite-файла или Railway Volume.
2. Замените файлы репозитория содержимым архива.
3. Выполните deployment.
4. Не удаляйте существующий файл базы: новые столбцы добавятся автоматически.

Важно: при использовании SQLite на Railway храните базу на подключённом Volume, иначе данные могут исчезнуть при новом deployment.


## Исправление v11.4

Добавлена совместимость со старыми SQLite-базами, где поля `results.pushups`, `results.pullups`, `results.squats` имеют ограничение `NOT NULL` без значений по умолчанию. Результаты теперь записываются одновременно в старые поля и в универсальную таблицу `result_values`. Удалять существующую базу не нужно.


## Исправление v11.4

- Бот больше не отвечает на обычные сообщения в группах.
- Удалено навязчивое предложение перейти в личный чат.
- В группе обрабатываются только зарегистрированные команды и inline-кнопки бота.

## Изменения v11.7

- Красивое подтверждение после сохранения: упражнения, баллы, место и серия.
- Кнопка «Изменить результат» после сохранения.
- При повторном вводе за выбранную дату показываются текущие значения.
- Централизованная проверка отрицательных и аномально больших значений.
- Незавершённый ввод хранится в SQLite в таблице `result_drafts`.
- После перезапуска Railway бот предлагает продолжить ввод или начать заново.
- Черновик удаляется только после успешного сохранения или явной отмены.
- Старая база и Railway Volume сохраняются; удалять БД не требуется.


## Что нового в v11.7

- В админском меню появилась кнопка «🟢 Активные челленджи».
- Администратор видит активные челленджи во всех группах, которыми управляет.
- Для каждого челленджа доступна карточка: даты, текущий этап, участники, отметившиеся сегодня, баллы, лидер и упражнения.
- Из карточки можно открыть рейтинг и правила, обновить данные или завершить челлендж.
- Завершение теперь требует отдельного подтверждения и не происходит случайно по одному нажатию.
- «Архив челленджей» показывает только завершённые сезоны, а не смешивает их с активным.
- В архив добавлено количество участников с результатами.
- Все новые callback-действия повторно проверяют права администратора.

База данных и Railway Volume совместимы с v11.5; удалять их не нужно.


## Изменения v11.7

- Исправлен поиск активного челленджа при вводе результата из личных сообщений.
- Если выбранная ранее группа не имеет активного сезона, бот ищет активные челленджи во всех группах пользователя.
- При одном доступном челлендже он выбирается автоматически.
- При нескольких активных челленджах бот предлагает выбрать нужный.
- Выбор защищён проверкой участия пользователя в группе.
