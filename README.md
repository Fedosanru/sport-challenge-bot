# Sport Challenge Bot v11.2

Telegram-бот для нескольких групп с универсальными упражнениями и отдельными правилами каждого челленджа.

## Что нового в v11.2

- Исправлено сохранение результатов: кнопка больше не молчит при потере FSM-состояния.
- Добавлена явная транзакция SQLite, WAL и ожидание блокировки базы для Railway.
- При повторном редактировании сохраняется полный набор упражнений, включая нули.
- Ошибки сохранения показываются пользователю и записываются в Railway Logs.
- После успешного сохранения кнопки подтверждения удаляются, чтобы исключить повторные нажатия.

## Возможности v11.1

- Кнопка `📋 Правила` в личном и групповом интерфейсе.
- Шаблоны правил:
  - классический — баллы пропорционально выполнению нормы;
  - выполнил / не выполнил — баллы только за полную норму;
  - ступенчатый — 50% нормы дают половину баллов, 100% дают все баллы;
  - собственные настройки.
- В собственных настройках выбираются:
  - способ начисления баллов;
  - учёт перевыполнения;
  - критерий успешного дня;
  - количество прошлых дней, доступных для исправления;
  - порядок вступления участников.
- Правила копируются вместе с челленджем.
- После создания условия фиксируются и не меняют старые результаты.
- Старая база v10/v11 мигрирует автоматически.

## Railway

Переменные:

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_PATH` — путь к SQLite, по умолчанию `sport_challenge.db`.
- `TIMEZONE` — по умолчанию `Europe/Moscow`.
- `ADMIN_IDS` — Telegram ID администраторов через запятую, необязательно.

Команда запуска уже указана в `Procfile`.

## Обновление

1. Сделайте резервную копию SQLite-файла или Railway Volume.
2. Замените файлы репозитория содержимым архива.
3. Выполните deployment.
4. Не удаляйте существующий файл базы: новые столбцы добавятся автоматически.

Важно: при использовании SQLite на Railway храните базу на подключённом Volume, иначе данные могут исчезнуть при новом deployment.
